JSON-LD Beispiel

Das JSON-LD Skript ist in der HTML Datei des 3D-Modells eingebettet, und kann von WebCrawlern indexiert werden.

Der Schlüsselwort @context gibt an, dass das JSON auf dem Schema.org-Vokabular basiert. Hier für ein 3D-Modell (@type).

Ein 3D-Model kann multiple Encodings vom Typ MediaObject haben. Es macht Sinn hier ein für das Web optimiertes Modell im Format "model/gltf-binary" (.glb) anzubieten, welches dann auch direkt im Browser gerendert werden kann. Dies dient hier als Input für das Schlüsselwort embedUrl. Da das Originalformat aber auch vom Format "model/gltf-binary" sein kann, wird dieses gezippt mit dem entsprechenden Encoding Format hier bereitgestellt (model/gltf-binary+zip). So können die Crawler zwischen dem optimierten Modell fürs Web und dem original Modell unterscheiden. Letzteres kann leicht die Ressourcen des Browsers überfordern, oder selbst die eines üblichen Office PCs. Ein Beispiel hierfür ist das Original vom Lincoln Memorial 3D-Modell.

Nicht berücksichtigt werden hier Kataloge von 3D Kacheln, wie wir sie z.B. bei digitalen Zwillingen haben. Dafür würde sich auch eher ein Schema.org DataCatalog eignen. Hier geht es zu den freien Daten des Digitalen Zwillings NRW - 3D-Gebäudemodell LoD2 im Scene Layer Package Format. Zur Anzeige der Tiles sind spezielle Viewer notwendig, wie z.B. der Digitale Zwilling NRW.

Die strukturierten Daten für Schema.org Datensätze (Dataset, DataCatalog, DataDownload) haben in der Praxis durchaus Relevanz. Diese werden für Google Dataset Search indexiert. Hier ein Beispiel für ein Dataset, das in der Google Dataset Search indexiert ist: Aggregierte Daten des maximal gemessenen Schalldrucks der Sensor Community
